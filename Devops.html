Let us know about Continuous Integration(CI):

Continuous Integration (CI) is a phase in the software development cycle where code from different team members or different features are integrated together. This usually involves merging code (integration), building the application and carrying out basic tests all within an ephemeral environment. CI workflows vary a lot depending on tools, programming language, project and many other factors but a common flow has these steps.
• Pushing to the code repository
• Static analysis
• Pre-deployment testing
• Packaging and deployment to the test environment
• Post-deployment testing

What is Source Control Mangement(SCM):

Source code management is the concept of storing the source code for your software applications in a centralised, managed place. It’s used by software development teams, who are the ones that gain the most benefit from it due to the multiple developers they have.
The main people involved in source code management are the developers or programmers. They are the ones that are working on the code, making changes, adding features and requirements.
Testers are also main users of source code. When they are required to test applications or features, they benefit from a source code management system and process, as it ensures they are working with accurate code and helps the team run better. GIT is used as one of the tool in SCM.

In SCM GIT, GITHUB and GITLAB is used to manage the Source Code

GIT: It is version control system to keep track of changes to files and projects over time.

GITHUB: It is a website the host GIT Repositories online, making it easier for developers to share their codes.

GITLAB: It is similar to GITHUB , GITLAB is a repository manager which lets teams collaborate on code.
